01 — 项目概述

项目背景与目标

向下滚动查看完整内容

在多人在线游戏市场中，桌游类产品因其社交属性和策略深度始终保持着稳定的用户需求。本项目以经典桌游 Monopoly（大富翁） 为灵感，打造了一款名为 "Vacashon" 的环球旅行主题多人在线棋盘游戏，支持 2–4 名玩家同时在线对局。玩家以世界各国城市为地产，通过掷骰子移动、购买地产、建造房屋、收取租金，目标是成为最后一个未破产的玩家，或在计时结束时拥有最高总资产。

项目由 8 人团队协作完成，开发周期约三个月，涵盖需求分析、技术选型、原型设计、功能开发、服务器部署、调试集成和最终交付等完整阶段。团队采用分组协作模式：游戏机制开发组（3 人）、服务器与网络组（2 人）、产品与交互设计组（2 人，本人所在组）、以及项目经理（1 人），各组并行推进并在关键节点进行联合集成。本人在项目中兼具产品、设计与技术三重角色，独立完成了从 Figma 原型设计到 Godot 引擎代码实现的全链路工作。

项目的核心目标包括以下几个方面：

第一，实现完整的多人在线游戏体验。基于客户端-服务器架构，通过 WebSocket 实现实时通信，支持多平台（桌面端、移动端、Web 端）玩家同时在线对局，服务器集中管理所有游戏逻辑以确保状态同步和公平性。

第二，构建标准化的游戏机制系统。完整还原大富翁核心玩法——掷骰子移动、购买地产、建造房屋/酒店、收取租金、事件卡牌（Lucky Suitcase / Mystery Passport）、海关（监狱）机制、破产判定，并加入计时赛制和总资产排名等创新规则。

第三，打造高质量的用户界面与交互体验。以 Figma 完成从低保真线框到高保真交互原型的三阶段迭代，再迁移至 Godot 引擎用 GDScript 实现全部界面功能，确保设计与游戏机制无缝衔接，同时适配多种屏幕尺寸。

第四，建立可靠的后端基础设施。使用 Python + FastAPI 搭建后端，PostgreSQL 管理动态数据，JSON 管理静态配置，Docker 容器化部署，Let's Encrypt SSL 加密通信，并配置自定义域名和专属邮件服务器支持账户注册与密码找回。

第五，以真实产品标准驱动团队协作。通过 Gantt 图规划 125+ 任务、Trello 看板跟踪进度、Git 版本控制、Discord 沟通协作，模拟真实产品团队的工作模式，从需求到交付走完产品全生命周期。

总体而言，本项目以一款完整的多人在线游戏产品为载体，贯穿了产品规划、交互设计、前端开发、后端架构和项目管理的全链路实践，是技术能力与产品思维的综合体现。

02 — 核心亮点

产品设计与功能亮点

项目在游戏机制、实时通信架构、UI/UX 设计流程、数据管理、跨平台部署和团队协作六大维度进行了深度实践。

向下滚动查看全部 6 个亮点
01
完整还原经典大富翁核心玩法并融入创新机制
游戏忠实还原了 Monopoly 的核心规则体系：掷骰子移动、购买城市地产、建造房屋提升租金、事件卡牌触发（Lucky Suitcase 和 Mystery Passport 两套牌组）、海关关押与释放机制、以及破产判定。

在此基础上，项目融入了多项创新设计：以环球旅行为主题，棋盘城市来自团队 8 名成员的母国；引入计时赛制，时间到后按总资产排名决胜；机场格子支持快速传送至下一个机场；所有游戏逻辑由服务器集中处理，客户端仅发送指令（如 {"action": "roll_dice"}），确保多人对局的公平性和一致性。
Monopoly 玩法 事件卡牌 计时赛制 服务器权威
02
客户端-服务器实时通信架构，支持跨平台多人在线
项目采用 Python + FastAPI 构建后端服务，结合两种通信协议：HTTP REST API 用于用户认证、资源加载等无状态请求；WebSocket 用于游戏内实时通信（移动、状态更新、回合切换），实现全双工低延迟交互。

后端架构清晰分层：GameHandler 处理 WebSocket 事件 → GameManager 验证回合权限并分发逻辑 → GameLogic 封装所有游戏规则 → 状态通过 Pydantic 序列化后广播给所有客户端。整套架构支持多房间并发运行，玩家可创建公开或私密房间，2–4 人同时在线对局。
FastAPI WebSocket REST API Pydantic 多房间并发
03
从 Figma 到 Godot 的完整 UI/UX 设计与实现流程
HCI 团队（本人所在组）采用 Figma 进行三阶段原型迭代：低保真线框 → 中保真布局 → 高保真交互原型，利用 Smart Animate 模拟骰子滚动、事件卡弹窗等关键微交互，并通过 Master Components 建立统一设计系统确保全局视觉一致性。

设计定稿后迁移至 Godot 引擎实现：每个界面作为独立 Scene 开发（模块化场景架构）；使用 Control / VBoxContainer / HBoxContainer 等节点实现响应式布局适配不同屏幕；弹窗交互通过独立 Scene 管理；所有动画和过渡效果均通过 GDScript + AnimationPlayer 在引擎内原生实现，无需外部工具。整个流程从零学习 Figma，独立完成了页面布局、视觉设计、配色方案、Logo 初稿、交互动效等全部 UI 工作。
Figma 原型 三阶段迭代 Godot 实现 响应式布局 GDScript 动画
04
静态配置与动态数据分离的数据管理策略
团队在数据管理上进行了充分的技术选型：先后构建了 FastAPI + SQLite 和 PostgreSQL 两套原型进行性能对比，最终选择 PostgreSQL 作为主数据库，因其支持对象存储、功能强大且安全可靠。

数据架构采用静态与动态分离策略：棋盘格子定义（PropertyCell、StreetCell、RailwaysCell 等）作为静态配置存储在 JSON 文件中，启动时加载；玩家账户、房间信息等动态数据存储在 PostgreSQL 数据库中，通过 SQLAlchemy ORM 操作。Pydantic 用于定义数据验证模式，保障前后端通信的数据结构和内容安全。
PostgreSQL SQLAlchemy JSON 配置 数据验证
05
容器化部署与完整的安全认证体系
项目基础设施完善：后端服务、Web 服务器和邮件服务器均通过 Docker 容器化部署；Nginx 托管游戏 Web 版本；Apache 作为反向代理路由 HTTPS 和 WebSocket 流量；Let's Encrypt SSL 证书加密所有通信；自定义域名 vacashon.online 配合 Poste.io 邮件服务器支持注册确认和密码重置邮件。

认证系统方面，实现了完整的用户注册 → 邮箱验证码确认 → 登录 → JWT Token 鉴权 → 忘记密码重置流程。所有受保护的 API 端点均需要用户认证，WebSocket 连接也通过 JWT 验证身份，确保实时通信的安全性。
Docker Nginx SSL/HTTPS JWT 认证 邮件服务
06
规范化的项目管理与多工具协同工作流
项目全程采用专业化的管理工具链：Gantt Project 规划 125+ 任务并设定 8 个关键里程碑（Alpha → Beta 0.1 → Beta 0.2 → 1.0 正式版）；Trello 看板按团队分类任务、设置截止日期和负责人，实时追踪进度；Git 进行版本控制和代码协同；Discord 按工作组划分频道进行日常沟通；Google Drive 管理文档、周报和工时表。

团队在开发过程中遇到了多次进度延迟和任务依赖冲突（如 Figma 设计未完成导致 UI 实现阻塞、数据库延迟影响动态显示功能），通过临时方案替代（如先用 JSON 模拟数据库）和灵活调整甘特图来保障项目整体进度，最终按时交付了功能完整的可玩产品。
Gantt 图 Trello Git 协同 敏捷调整 里程碑管理

04 — 项目成果与总结

成果与收获

向下滚动查看完整内容

经过三个多月的团队协作，项目最终按期交付了名为 "Vacashon" 的完整多人在线棋盘游戏产品，实现了从概念设计到可运行产品的全流程落地。

产品交付成果方面，游戏包含完整的账户系统（注册、登录、邮箱验证、密码找回）；支持创建公开/私密房间并选择角色Token的大厅系统；完整的大富翁核心玩法——掷骰子移动、购买地产、建造房屋、收取租金、事件卡牌、海关机制、破产判定和计时赛制；全套 UI 界面覆盖主菜单、规则说明、设置面板、反馈系统和游戏内操作界面；以及沉浸式的音乐和音效系统，支持独立音量调节且偏好设置自动持久化。

个人贡献方面，作为 HCI 团队核心成员，本人主导了 UI 原型设计的全流程：从零学习 Figma 并独立完成了完整的高保真交互原型，涵盖页面布局规划、屏幕构图设计、玩家卡片与交互按钮的视觉设计、棋盘界面样式、Token 选择与玩家数量选项界面、统一配色方案、Logo 初稿设计（借助 AI 工具辅助）以及交互过渡动效的设计。随后在 Godot 中完成了全部技术实现：使用节点系统搭建页面结构并适配响应式布局，通过 GDScript 开发动画效果和弹窗行为，实现了按钮操作、场景切换、条件显示（如私密房间的密码输入框）等核心 UI 功能，并持续优化代码确保用户交互与设计意图的一致性。在开发后期，与后端和游戏逻辑团队紧密合作，确保 HCI 模块顺利集成到整体系统中。总计投入约 125 小时。

技术能力提升方面，通过本项目快速掌握了 Figma 原型设计工具和 Godot 游戏引擎的 UI 开发能力，积累了从设计稿到工程实现的完整经验。深入理解了客户端-服务器架构下的实时通信机制（WebSocket + REST API），以及多人在线游戏的状态同步原理。同时在 PostgreSQL 数据库设计、Docker 容器化部署、Git 多分支协作等方面也获得了实践经验。

项目管理与协作能力方面，亲身经历了 125+ 任务的甘特图规划和 8 个里程碑版本的迭代发布过程，深刻理解了软件项目中进度管理、任务依赖和风险应对的重要性。在跨组协作中，学会了在设计未定稿时用临时方案推进开发、在多分支合并冲突时快速定位和解决问题，以及在不同团队之间对齐需求和同步进度的沟通技巧。

总体而言，本项目不仅是一次综合性的技术实践，更是一次接近真实工作环境的团队协作训练。虽然受限于时间，部分计划功能（如摄像机跟随系统、新手教程、多主题背景、AI 机器人玩家等）未能最终实现，但团队在有限时间内交付了功能完整、界面精美、可多人在线游玩的产品，这一成果是所有成员共同努力的结果。
